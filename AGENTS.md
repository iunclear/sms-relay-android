# 交付闭环

对本仓库每次需要改动文件的任务，除非用户明确要求不提交、不推送、不发布或只做本地检查，否则必须完成以下闭环后才能结束：

1. 完成与改动相关的本地构建、测试、Lint 和清单检查。
2. 只暂存本次任务的文件，创建清晰的 Git 提交并推送当前目标分支（默认 `main`）。
3. 找到由该提交触发的 GitHub Actions，持续检查直到完成。
4. 若构建或发布失败，读取失败日志，修复问题，然后从本地验证、提交、推送开始重复该循环。
5. 仅在 CI 成功后报告完成；有 APK 或其他工件时，确认其已上传并报告提交号、运行链接和工件名称。

不要把“已提交”或“已触发 CI”视为完成。完成标准是由本次提交触发的构建成功。
